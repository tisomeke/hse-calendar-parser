```markdown
# ТЕХНИЧЕСКОЕ ЗАДАНИЕ

## Парсер вузовского расписания ФГН НИУ ВШЭ (НН) → iCalendar (.ics)

**CLI-инструмент с модульной архитектурой для будущего масштабирования на Telegram-бота**

---

## 1. Цель и контекст

**Цель:** Разработать консольное Python-приложение, которое принимает на вход вузовский Excel-файл с расписанием (экспорт из Google Sheets) и генерирует корректный `.ics`-файл (iCalendar RFC 5545) для выбранной студенческой группы.

**Контекст использования:** Студент получает от ВШЭ расписание в виде `.xlsx`. Таблица имеет сложную многоуровневую структуру, нестандартную разметку внутри ячеек, отмены, восстановления, недельную чередовку и смешение нескольких дисциплин в одной ячейке. Вузовское приложение неудобно, а стандартные календари (Google Calendar, Apple Calendar, Outlook) не умеют напрямую читать такие таблицы.

**Ограничение:** Решение — исключительно CLI. Никаких веб-интерфейсов, баз данных и Telegram-ботов в этом ТЗ. Но архитектура должна быть модульной, чтобы в будущем можно было заменить CLI-ввод на HTTP/Telegram-ввод без переписывания ядра.

---

## 2. Входные данные

### 2.1. Файл

- Формат: `.xlsx` (Excel 2007+, OpenXML)
- Источник: экспорт из Google Sheets
- Один файл соответствует **одному модулю** (3-й или 4-й)
- В файле содержится несколько **блоков** (не отдельных листов, а секций на одном листе), разделённых заголовками:
  - `# 1 курс`
  - `# 2 курс`
  - `# 3 курс`
  - `# 4 курс`
  - `# Календарь` (вспомогательная таблица, дублирующая периоды модулей)

### 2.2. Структура блока курса

Каждый блок — это таблица со следующей структурой:

| День | Время | Группа 1 Предмет | Группа 1 Ауд. | Группа 1 Корпус | Группа 2 Предмет | ... |
|------|-------|------------------|---------------|-----------------|------------------|-----|
| понедельник | 08:00-09:20 | ... | ... | ... | ... | ... |

- **Строка 1–2:** Заголовок с указанием курса и периода модуля. Пример: `БАКАЛАВРИАТ - 1 курс, 3 модуль (09.01. - 24.03.)`
- **Строка 3–7:** Легенда корпусов (БП, К, Л, Р, С)
- **Строка 8:** Повтор заголовка или ссылки на online
- **Строка 9:** Заголовки столбцов: `День`, `Время`, далее тройки `Предмет | Ауд. | Корпус` для каждой группы
- **Далее:** собственно расписание

**Группы и их коды:**

| Курс | Группы |
|------|--------|
| 1 курс | `25ФИЛ1`, `25ФИЛ2`, `25ФПЛ1`, `25ФПЛ2` |
| 2 курс | `24ФИЛ1`, `24ФПЛ1`, `24ФПЛ2` |
| 3 курс | `23ФИЛ1`, `23ФИЛ2`, `23ФПЛ1`, `23ФПЛ2` |
| 4 курс | `22ФИЛ1`, `22ФИЛ2`, `22ФПЛ1`, `22ФПЛ2` |

### 2.3. Содержимое ячеек «Предмет»

Ячейка — это **многострочный текстовый блок**, а не однозначная структура. Один блок может описывать несколько занятий.

**Пример 1: Конкретные даты + тип + разделитель**

```
05.11, 19.11, 03.12, 17.12
___________________________
Ключевые тексты русской литературы
Гельфонд М. М.
```

**Пример 2: Несколько занятий в одной ячейке с разными датами**

```
05.11, 19.11, 03.12, 17.12 -лекция
______________________________
12.11, 26.11, 10.12 - семинар гр.1
Старославянский язык и историческая грамматика русского языка - лекция
Мохова Л.К.
```

**Пример 3: Период + отмена**

```
с 12.01 по 23.03
Античная литература
Перевезенцева А. Ю.
12.01 - отмена
```

**Пример 4: Отмена с восстановлением**

```
29.04 - отмена (восстановлено 02.04)
Второй иностранный язык (французский)
Келлерман А. А.
```

**Пример 5: Недельная чередовка**

```
верхняя неделя
с 10.01, 24.01-отмена
Теория языка I (семинары)
Бинштейн М. М.
```

**Пример 6: Несколько параллельных дисциплин**

```
Второй иностранный язык (французский)
Келлерман А. А.
Второй иностранный язык (немецкий)
Сибирцева В.Г.
```

**Пример 7: Ссылка на внешнее расписание**

```
Занятия проводятся по расписанию дисциплины
Английский язык для общих коммуникативных целей:
начальный курс, базовый курс, основной курс, продвинутый курс
```

**Пример 8: Физкультура**

```
Занятия по дисциплине "Физическая культура" проводятся согласно расписанию секций
```

**Пример 9: Восстановление вне периода**

```
25 мая восстановление
МКД Теория литературы
Большухин Л.Ю.
```

**Пример 10: Перенос аудитории для конкретной даты**

```
13.06 (за 12.06)
Немецкий язык
Леонова С.С.
```

### 2.4. Ячейки «Ауд.» и «Корпус»

- Могут содержать номер: `102`, `О12`, `.002`, `ОО2`, `ОО3`
- Могут содержать `online`, `онлайн`, `олайн` (sic!)
- Могут содержать уточнения: `online<br>102 ауд. для трансляции`
- Могут быть пустыми

## 3. Выходные данные

### 3.1. Формат: iCalendar (.ics)

- Стандарт: RFC 5545
- Кодировка: UTF-8
- Каждое занятие — отдельный компонент `VEVENT` (не `RRULE`, т.к. даты нерегулярны)
- Продолжительность события определяется из столбца `Время`

### 3.2. Поля VEVENT

| Поле | Значение | Пример |
|------|----------|--------|
| `UID` | Уникальный ID | `hse-25fpl1-2025-11-05-11-10-3f4a9c` |
| `DTSTART` | Дата и время начала | `20251105T111000` |
| `DTEND` | Дата и время окончания | `20251105T123000` |
| `SUMMARY` | `[Тип] Название дисциплины` | `[Лекция] Теория языка I` |
| `LOCATION` | `Ауд. XXX, Корпус YYY` или `Online` | `Ауд. 102, Корпус К` |
| `DESCRIPTION` | ФИО преподавателя + исходная строка | `Преподаватель: Бинштейн М.М.\nИсточник: ...` |
| `STATUS` | `CONFIRMED` | `CONFIRMED` |

**Примечание:** Отменённые занятия (`отмена`) **не включаются** в `.ics` (не создаём событие), за исключением случаев, когда отмена сопровождается восстановлением на другую дату — тогда создаём событие на дату восстановления.

## 4. Архитектура системы

```
┌─────────────────────────────────────────────┐
│           Входной интерфейс (CLI)            │
│  click / argparse: файл, группа, параметры   │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────▼──────────────────────────┐
│         Слой оркестрации (Controller)       │
│  Валидация входных данных, выбор курса,     │
│  запуск pipeline, сборка отчёта             │
└──────────────────┬──────────────────────────┘
                   │
    ┌──────────────┼──────────────┐
    ▼              ▼              ▼
┌────────┐   ┌──────────┐   ┌──────────┐
│ Reader │   │  Parser  │   │ DateEngine│
│(openpyxl)│  │(NLP/rules)│  │(calendar) │
└────────┘   └──────────┘   └──────────┘
    │              │              │
    └──────────────┼──────────────┘
                   ▼
        ┌──────────────────┐
        │ Internal Model   │
        │ (dataclasses)    │
        │ List[Event]      │
        └────────┬─────────┘
                 ▼
        ┌──────────────────┐
        │   ICS Exporter   │
        │   (icalendar)    │
        └────────┬─────────┘
                 ▼
        ┌──────────────────┐
        │   Logger/Report  │
        │   (warnings)     │
        └──────────────────┘
```

### 4.1. Модули и их ответственность

| Модуль | Ответственность | Библиотека |
|--------|----------------|------------|
| `reader` | Чтение `.xlsx`, поиск блока курса, определение смещений столбцов группы, извлечение периода модуля из заголовка | `openpyxl` |
| `parser` | Разбор многострочного текста ячеек на структурированные токены | `re` (regex), кастомные правила |
| `date_engine` | Генерация конкретных дат из абстрактных описаний | `datetime`, `dateutil` |
| `models` | Pydantic/dataclasses для внутреннего представления | `pydantic` или `dataclasses` |
| `exporter` | Преобразование `List[Event]` в `icalendar.Calendar` | `icalendar` |
| `controller` | Склейка модулей в pipeline, обработка ошибок, формирование отчёта |
| `cli` | Интерфейс командной строки | `click` |
| `utils` | Хелперы: нормализация текста, транслитерация групп в ID, хеширование для UID |

## 5. Детальные требования к парсингу

### 5.1. Этап 1: Чтение файла (Reader)

**R-1.1.** Программа должна корректно открывать `.xlsx` через `openpyxl` в режиме `read_only=True` для экономии памяти.

**R-1.2.** Автоопределение курса по коду группы:
- `25` → 1 курс
- `24` → 2 курс
- `23` → 3 курс
- `22` → 4 курс

Если автоопределение невозможно — программа требует явного указания курса через флаг `--course`.

**R-1.3.** Поиск блока курса в файле:
- Блоки разделены текстом вроде `# 1 курс`, `# 2 курс` и т.д.
- Искать по всем строкам листа первое вхождение заголовка, содержащего `<N> курс`
- Блок заканчивается следующим заголовком курса, заголовком `# Календарь`, или концом данных

**R-1.4.** Извлечение периода модуля:
- Из заголовка блока извлекается строка вида `3 модуль (09.01. - 24.03.)`
- Формат дат: `ДД.ММ.` (без года)

**Правило определения года:**
- 3 модуль (январь–март) → `year = academic_year_start + 1`
- 4 модуль (апрель–июнь) → `year = academic_year_start + 1`
- 1 модуль (сентябрь–декабрь) → `year = academic_year_start`
- 2 модуль (ноябрь–декабрь) → `year = academic_year_start`

Параметр `--academic-year-start` (по умолчанию 2025).

**R-1.5.** Определение столбцов группы:
- В строке заголовков (~9-я строка блока) ищем точное совпадение кода группы
- После нахождения: `Предмет` = столбец группы, `Ауд.` = столбец+1, `Корпус` = столбец+2
- Если группа не найдена — ошибка с перечислением найденных групп

**R-1.6.** Чтение строк расписания:
- Итерируем строки от заголовков до конца блока
- Собираем: `День` (столбец A), `Время` (столбец B), `Предмет`, `Ауд.`, `Корпус`
- День недели может тянуться на несколько строк (merged cells или просто повторение). Если пустая — наследуем сверху.
- Время в формате `HH:MM-HH:MM` или пустое (пропускаем)

### 5.2. Этап 2: Парсинг ячеек (Parser)

**R-2.1. Нормализация входного текста:**
- Удаление HTML-тегов (`<br>` → `\n`)
- Замена `&nbsp;` на пробел
- Нормализация переносов: `\r\n` → `\n`
- Удаление лишних пробелов в начале/конце строк
- Нормализация `online`/`онлайн`/`олайн`/`ONLINE` → `online`
- Нормализация `отмена`/`Отмена`/`ОТМЕНА` → `отмена`
- Нормализация `восстановление`/`восстановлено`/`восстнолвение` (sic!) → `восстановление`

**R-2.2. Разделение ячейки на логические блоки:**
Разделители:
- Линия из подчёркиваний: `____+` (≥10 символов `_` или `-`)
- Двойной перенос строки: `\n\n`
- Тег `<br><br>` (уже нормализован)

**R-2.3. Извлечение дат из текстового блока:**

| Паттерн | Пример | Результат |
|---------|--------|-----------|
| Конкретные даты | `03.11, 17.11, 01.12` | Список дат |
| Период | `с 03.11 по 15.12` | Все даты периода, совпадающие с днём недели |
| Конкретные + тип | `05.11, 19.11 -лекция` | Даты + тип |
| Отмена | `12.01 - отмена` | Исключить дату |
| Отмена с восстановлением | `29.04 - отмена (восстановлено 02.04)` | Исключить 29.04, добавить 02.04 |
| Восстановление | `25 мая восстановление` | Добавить дату |
| Недельная чередовка | `верхняя неделя` | Флаг чётности |

**R-2.4. Определение типа занятия:**
- `лекция`, `семинар`, `практикум`, `НИС`, `МКД`, `МДК`, `консультация`
- Если не найден — по умолчанию `занятие`

**R-2.5. Извлечение названия дисциплины:**
- Первая осмысленная строка после удаления дат и типа
- Префиксы `МКД`/`МДК`/`НИС` переносятся в тип

**R-2.6. Извлечение преподавателя(ей):**
- После названия дисциплины
- Паттерн: `Фамилия И.О.` или `Фамилия Имя Отчество`
- Несколько преподавателей через запятую

**R-2.7. Обработка подгрупп:**
- Флаг `--subgroup 1` или `--subgroup 2`
- Блоки с `гр.X`, где X ≠ подгруппа — игнорируются
- Блоки без указания подгруппы — включаются (общие занятия)
- Если подгруппа не указана, но в ячейке есть разделение — warning и пропуск

**R-2.8. Обработка специальных случаев:**

| Текст | Действие |
|-------|----------|
| `Занятия проводятся по расписанию дисциплины Английский язык...` | **SKIP**, лог: «Пропущено: внешнее расписание (Английский)» |
| `Занятия по дисциплине "Физическая культура"...` | **SKIP**, лог: «Пропущено: Физическая культура» |
| `Занятия по дисциплинам дополнительного профиля MINOR` | **SKIP** |
| `ССЫЛКИ НА ONLINE - ЗАНЯТИЯ` | **SKIP** |
| Изолированная фамилия (`Леонова`, `Королева М.В.`) | **SKIP** |

**R-2.9. Обработка аудиторий и корпусов:**
- Базовые значения из столбцов `Ауд.` и `Корпус`
- Уточнения в тексте ячейки перекрывают базовое для конкретной даты
- Корпус = `online` → `LOCATION = Online`

### 5.3. Этап 3: Генерация дат (DateEngine)

**R-3.1.** Генерация дат из периода: все даты между `start_date` и `end_date`, совпадающие с днём недели.

**R-3.2.** Фильтрация по конкретным датам: если указаны конкретные даты — используем только их, период игнорируется.

**R-3.3.** Фильтрация по недельной чередовке:
- `week_number = ((date - start_date).days // 7) + 1`
- `верхняя неделя` → нечётные `week_number`
- `нижняя неделя` → чётные `week_number`

**R-3.4.** Восстановления добавляются вне зависимости от дня недели.

**R-3.5.** Даты без года: формат `ДД.ММ` или `ДД месяц_на_русском`. Месяцы: января, февраля, марта, апреля, мая, июня, июля, августа, сентября, октября, ноября, декабря.

### 5.4. Этап 4: Внутреннее представление (Models)

```python
class Event:
    uid: str
    summary: str          # [Тип] Название
    start: datetime       # timezone-aware, Europe/Moscow
    end: datetime
    location: str | None
    description: str
    source_text: str      # Исходный текст ячейки
    status: "CONFIRMED"

class RawLessonBlock:
    dates: list[date]
    title: str
    lesson_type: str
    teachers: list[str]
    subgroup: int | None
    is_online: bool
    is_cancelled: bool
    recovery_date: date | None
    week_parity: "upper" | "lower" | None
    location_override: dict[date, str]
```

### 5.5. Этап 5: Экспорт (Exporter)

**R-5.1.** Библиотека: `icalendar`

**R-5.2.** Каждый `Event` → `icalendar.Event` с полями `UID`, `DTSTART`, `DTEND`, `SUMMARY`, `LOCATION`, `DESCRIPTION`, `STATUS`

**R-5.3.** Timezone: `Europe/Moscow`

**R-5.4.** Заголовок `.ics`:
```
BEGIN:VCALENDAR
VERSION:2.0
PRODID:-//HSE Schedule Parser//RU
CALSCALE:GREGORIAN
METHOD:PUBLISH
X-WR-CALNAME:Расписание <код_группы>
X-WR-TIMEZONE:Europe/Moscow
```

**R-5.5.** UID: `hse-<group>-<YYYY-MM-DD>-<HHMM>-<hash(title+teacher)>`

## 6. CLI-интерфейс

### 6.1. Команда и аргументы

```bash
python -m hse_parser \
  --file "3 модуль.xlsx" \
  --group 25ФПЛ1 \
  --output schedule.ics \
  [--subgroup 1] \
  [--academic-year-start 2025] \
  [--skip-minor] \
  [--skip-english] \
  [--skip-pe] \
  [--verbose]
```

| Аргумент | Обязательный | Описание |
|----------|--------------|----------|
| `--file`, `-f` | Да | Путь к `.xlsx` |
| `--group`, `-g` | Да | Код группы (регистронезависимый) |
| `--output`, `-o` | Нет | Путь выходного файла (по умолчанию: `<group>_module<N>.ics`) |
| `--subgroup` | Нет | Номер подгруппы (1 или 2) |
| `--academic-year-start` | Нет | Год начала учебного года (по умолчанию 2025) |
| `--skip-minor` | Нет | Пропускать занятия MINOR (по умолчанию True) |
| `--skip-english` | Нет | Пропускать английский (по умолчанию True) |
| `--skip-pe` | Нет | Пропускать физкультуру (по умолчанию True) |
| `--verbose`, `-v` | Нет | Подробный вывод в консоль |

### 6.2. Вывод в консоль

- При `--verbose`: прогресс по строкам
- Итоговая статистика: `Создано N событий, пропущено M ячеек, W warnings`
- Список проблемных ячеек

## 7. Логирование и отчёт об ошибках

**R-7.1.** Все пропущенные/нераспознанные ячейки записываются в `warnings` с указанием:
- Исходного текста (truncated до 200 символов)
- Причины: `UNPARSEABLE_DATES`, `EXTERNAL_SCHEDULE`, `MULTIPLE_SUBGROUPS_UNSPECIFIED`, `EMPTY_CELL`, `UNKNOWN_FORMAT`
- Номера строки/столбца в исходном Excel

**R-7.2.** Итоговый отчёт:
```
=== ОТЧЁТ ===
Всего обработано ячеек: 120
Успешно распарсено: 87
Пропущено: 33
  - Внешнее расписание (английский): 15
  - Физкультура: 10
  - Нераспознанный формат: 5
  - Подгруппа не указана: 3

Сохранено в: 25ФПЛ1_module3.ics
Событий в календаре: 94
```

**R-7.3.** Если `warnings` содержат `UNKNOWN_FORMAT` или `MULTIPLE_SUBGROUPS_UNSPECIFIED` — exit code `0`, но в stderr: `ВНИМАНИЕ: Обнаружены нераспознанные ячейки. Проверьте отчёт выше.`

## 8. Масштабируемость (интерфейсы для Telegram-бота)

Код должен быть структурирован так, чтобы в будущем можно было:

1. **Заменить `cli.py`** на `telegram_bot.py` без изменения `controller.py`
2. **Использовать `controller.pipeline()`**:
   ```python
   def run_pipeline(file_bytes: bytes, config: ParseConfig) -> ParseResult:
       # Возвращает .ics bytes + отчёт
   ```
3. **`ParseConfig`** — Pydantic-модель всех параметров
4. **`ParseResult`** — возвращает:
   - `ics_content: bytes`
   - `report: str`
   - `warnings: list[Warning]`
   - `events_count: int`

## 12. Примечания для AI-агента

1. **Не пытайся угадать всё.** Если ячейка не поддаётся парсингу — пропусти с warning.
2. **Русский язык — первый класс.** Все строки, дни недели, месяца — на русском. Regex Unicode-friendly.
3. **HTML в Excel.** Обрабатывай `<br>` и `&nbsp;` до regex.
4. **Год — ловушка.** В таблице нет года. Определяй по модулю и `--academic-year-start`.
5. **Модульность важнее скорости.** Чёткие границы между модулями — ключ к масштабированию.
6. **Не используй LLM внутри парсера.** Только детерминированные regex и правила.
