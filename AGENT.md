# AGENT.md — HSE Calendar Parser

## О проекте
Парсер расписания ФГН НИУ ВШЭ (Нижний Новгород) из Excel (.xlsx) в iCalendar (.ics).
Два пакета: `hse_parser/` (ядро) и `hse_schedule_parser/` (TUI-интерфейс).

---

## Skill: Тестирование нового расписания

### Когда использовать
Пользователь сказал: "новый тип расписания", "надо протестить", "проверь файл", "сделай ics и проверь", или принёс незнакомый .xlsx файл.

### Процесс

#### Шаг 1: Изучить файл
1. Открыть .xlsx через openpyxl и посмотреть структуру листов
2. Определить: какие колонки есть, где группа, где время, где дисциплина
3. Сравнить с существующим `table.local.xlsx` — понять отличия

#### Шаг 2: Запустить парсинг
```bash
# Через CLI (быстро)
python -m hse_parser.cli --file "путь/к/файлу.xlsx" --group ГРУППА --output test.ics --verbose

# Через TUI (интерактивно)
python -m hse_schedule_parser
```

#### Шаг 3: Проверить результат
1. Прочитать отчёт в stdout — сколько ячеек распарсено, сколько пропущено
2. Открыть .ics файл — проверить, что события выглядят корректно
3. Посмотреть warnings — если есть `UNPARSEABLE_FORMAT`, разобраться с форматом ячеек

#### Шаг 4: Если парсинг сломался
1. Найти проблемную ячейку в Excel
2. Посмотреть её содержимое
3. Определить, какой компонент парсера отвечает за этот формат:
   - `hse_parser/parser/cell_parser.py` — разбор содержимого ячейки на блоки
   - `hse_parser/parser/date_parser.py` — извлечение дат из текста
   - `hse_parser/parser/rules.py` — правила пропуска (английский, физра, MINOR)
   - `hse_parser/parser/text_cleaner.py` — нормализация текста
   - `hse_parser/parser/line_filters.py` — фильтрация строк внутри ячейки
4. Добавить поддержку нового формата в соответствующий компонент
5. Написать тест в `tests/`
6. Запустить `pytest -v` — убедиться, что старые тесты проходят

#### Шаг 5: Сверить корректность
1. Открыть .ics в текстовом редакторе — проверить структуру событий
2. Убедиться, что:
   - Даты соответствуют расписанию
   - Время начала/конца корректно
   - Аудитории указаны
   - Нет дубликатов событий
   - Отменённые занятия (зачёркнутый текст) не попали в календарь

### Важные нюансы
- Формат дат ВШЭ: "ДД.ММ" или "ДД.ММ — ДД.ММ"
- Недели: верхняя (числитель) / нижняя (знаменатель)
- Зачёркнутый текст = отмена занятия
- Online-занятия помечены `Online`
- Английский, физкультура и MINOR пропускаются по умолчанию (настраивается)
- Тестовые данные лежат в `tests/fixtures/`
- Пример рабочего файла: `table.local.xlsx`

## Skill: Добавление нового формата даты

### Когда использовать
Пользователь просит поддержать новый формат даты в расписании.

### Процесс
1. Открыть `hse_parser/parser/date_parser.py`
2. Добавить новый regex-паттерн в `DATE_PATTERNS`
3. Написать тест в `tests/test_date_engine.py`
4. Запустить `pytest tests/test_date_engine.py -v`
5. Проверить, что старые форматы не сломались

## Skill: Добавление нового шага в TUI wizard

### Когда использовать
Пользователь просит добавить новый шаг (экран) в интерактивный мастер.

### Процесс
1. Открыть `hse_schedule_parser/wizard.py`
2. Создать новый класс шага, наследуясь от `WizardStep`
3. Добавить шаг в `WizardState.steps` в нужном порядке
4. Обновить `run_wizard()` если нужно
5. Запустить `python -m hse_schedule_parser` — проверить навигацию

## Skill: Добавление новой команды в CLI

### Когда использовать
Пользователь просит добавить новую CLI-команду или опцию.

### Процесс
1. Открыть `hse_parser/cli.py`
2. Добавить `@click.option(...)` или новую `@click.command()`
3. Добавить поле в `ParseConfig` в `hse_parser/config.py`
4. Передать новую опцию в `run_pipeline()` в `controller.py`
5. Запустить `python -m hse_parser.cli --help` — проверить
